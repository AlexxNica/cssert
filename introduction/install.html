cssert – CSS Unit testing

Make sure your styles don't change when your files do.

When working on a large project, you sometimes need to refactor your CSS. If you aren't constantly aware of all the different dependencies, you can end up altering the styles somewhere unexpected. This framework is designed to help you create key test cases linked to your styles. If the tests pass after refactoring, you can be sure your design is intact.

Design

This works by generating a simple HTML skeleton unit which matches the minimal page structure around the element whose styles you want to test. This unit contains references to your project's CSS. You can then refactor your CSS as much as you like and as long as the styles amount to the same visual appearance, the test will continue to pass.

Usage

The easiest way to create test cases is to install the bookmarklet. To generate a test unit, load a page whose styles you want to maintain then run the bookmarklet. If you click on an element, the test case will be generated in the floating panel. Copy and paste this into the test section of the standard test page. Open that page in a browser now and all the tests should pass. You can now refactor your CSS and the tests will let you know if something has changed unexpectedly.

Limitations

  • If your HTML changes, your test cases are no longer valid. A potential future direction for this project may be to simplify automatic test case generation tied into a VCS or build system
  • The HTML unit in the generated test units don't integrate JS but the measured styles do. This means that if you rely on JS setting styles, the test will probably fail.
